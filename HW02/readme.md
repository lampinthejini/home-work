**Last Updated:** 2026-07-13

## 실행 방법

### 1. 실행 전 준비
- `python -m venv .venv` 로 가상환경을 만듭니다.
- `.venv\Scripts\activate` 로 가상환경을 켭니다. (macOS/Linux: `source .venv/bin/activate`)
- `pip install -r requirements.txt` 로 필요한 패키지를 설치합니다.
- `.env.example` 파일을 복사해서 `.env` 파일을 만듭니다. (`cp .env.example .env`)
- Docker Desktop이 설치되어 실행 중이어야 합니다. (컨테이너 자체는 아래 2번에서 자동으로 만들어집니다)

### 2. run_desktop.py로 실행하기 (추천)
```
python run_desktop.py
```
- `hw02-postgres` 도커 컨테이너가 없으면 자동으로 만들고, 꺼져 있으면 자동으로 켭니다.
- 데이터베이스 마이그레이션(Alembic)을 자동으로 최신 상태까지 적용합니다.
- 데스크톱 창이 뜨면서 바로 서비스를 사용할 수 있습니다.

### 3. 서버만 직접 열어서 시작하기
데스크톱 창 없이 웹 서버만 쓰고 싶을 때 사용합니다.

1. 도커 컨테이너를 먼저 준비합니다. (`run_desktop.py`를 한 번도 실행한 적이 없다면 아래 명령으로 직접 만들어주세요. 이미 만들어져 있다면 건너뛰어도 됩니다.)
   ```
   docker run -d --name hw02-postgres -e POSTGRES_USER=user -e POSTGRES_PASSWORD=password -e POSTGRES_DB=contact_db -p 5432:5432 postgres:16
   ```
2. 서버를 실행합니다.
   ```
   uvicorn contact_app.main:app --reload
   ```
3. 브라우저에서 http://127.0.0.1:8000 으로 접속합니다.

## 느낀점

이번 과제를 통해서 AI를 활용한 바이브코딩에 대한 개념을 이해하고, TRD를 작성하면서 비교적 쉽게 프로그램을 만드는 과정을 배웠습니다. 그 과정에서 좋았던 점도 있고, 아쉬웠던 점도 있었으며, 동시에 내가 채워나가야 할 부분, 그리고 난해했던 부분에 대해서 정리해서 올리려고 합니다.

1. 좋았던 점.
 바이브코딩에 대한 개념을 배울 수 있어서 1차적으로 좋았습니다. 그리고 프로그램이 만들어지는 과정을 에이전트, 스킬을 활용하면서 효율적으로 진행할 수 있었고, AI와 함께 소통하며, 세세한 기획을 짜서, 좋은 프로그램이 나왔던 거 같습니다.기본적으로 PRD와 화면정의서, 기술정의서, 과제 개요가 잘나왔기에 이를 잘 활용할 수 있었으며, 이에 대한 자료를 기반으로 했기에 어려운 부분들을 많이 개선할 수 있었다고 생각합니다. 그리고 빠른 시간으로 제작이 되다보니까, 실질적으로 상용할 때, 어떤 느낌일지에 대해서 생각해보았고, 그 결과, 제작하지 않는 사람들에게는 도커를 열고 컨테이너를 사용하는 것에 어려움을 느낄 수 있을 거라 생각이 되어서, 바로 실행할 수 있는 파일을 만들고, 도커를 따로 키지 않은 경우, 도커를 킬 수 있게끔 제작했습니다.이 모든 과정이 배움이었고, 더 나아갈 수 있는 하나의 과정이라는 사실이 너무 기뻤습니다. 단순한 과제로 끝내는 것이 아니라, 실무에서 어떻게 적용할지, 혹은 다른 부분에 대해서 바이브코딩을 했을 때, 어떤 방식으로 진행해야할지에 대한 실마리가 잡힌 것 같아서 기분이 좋았습니다. 아이디어와 기획력이 있다면, 충분히 다른 프로그램을 만들 수 있을 것 같다는 자신감이 생겼습니다. (물론 이를 점검할 수 있는 실력과 능력을 기르는 것이 중요하다고 느꼈지만요!)

2. 아쉬웠던 점.
이와는 별개로 제가 코딩 실력이 늘었냐 하면, 그것은 아닐거라고 생각이 들었습니다. 아직 이해력이 많이 부족해서, 쭉 검토해나가면서 배우려고 했지만, 제대로 흡수를 하지 못했다는 생각이 들고, 만약 백지상태에서 이를 만들어낼 수 있을까? 라는 질문에 물론 클로드코드, LLM의 도움을 받으면 할 수 있다고는 생각하지만, 지금과 같은 상황과는 다르게 만일 LLM이 명령을 따라주지 않거나, 기능상 오류가 잘 고쳐지지 않을 때, 어떻게 해야할지에 대한 대처능력은 아직 길러지지 않았다고 생각했습니다.필요시에는 내가 직접 개입을 해야한다고 생각하나, 설령 개입을 하더라도, 어떤 명령을 내릴지, 어떤 부분이 정확히 문제인지 파악이 어렵다는 것을 생각했습니다.이부분에 대해서는 좀 더 검토하고, 웹,데이터베이스, 파이썬에 대해서 교육받은 내용을 다시 한번 복습하고, 더불어 배우지 않은 부분이더라도, 좀 더 파고 들어서, 확실히 이해할 수 있는 과정을 거쳐야 할 거 같다고 생각하며, 과제는 실제로 끝났어도 아직 끝나지 않았다는 기분이 들었습니다.

3. 내가 채워나가야 할 부분
위에도 말했듯이 AI가 많은 부분을 차지했고, 실제 선생님께서 제시한 프로그램을 기반으로 만들어진 부분이기 때문에, 기본적인 개념에서 아직 부족하다고 느꼈습니다. 특히 FASTAPI와 라우터에 대한 개념, 그리고 CRUD를 어떻게 상호작용하는 부분에 대한 충분한 이해, 마지막으로 웹에 대한 부분이 정확히 어떻게 상호작용하고 굴러가는지, 단순한 개념이 아니라, 흘러가는 흐름을 이해하고, 정확한 흐름으로 갈 수 있게끔 유도하는 능력을 가져야한다고 생각했습니다.기본적인 개념을 알아가고 완벽하게 구현하는 것까지는 바라지 않습니다. 적어도 내가 코드들을 검토하면서, 그 코드가 왜 있고, 왜 있어야 하는지, 그리고 이를 더 좋은 방안으로 같이 고민할 수 있을정도로 AI와 소통을 하는 것 또한, 필요시 선생님께 질문을 하는 것으로 계속 채워나가야 한다고 생각했습니다.

4. 난해했던 부분
 실제 연락처 프로그램과 과정은 같을지 몰라도, 편의성 부분에 대해서 생각했을 때, 많이 다르다고 생각했습니다. 핸드폰에 있는 연락처프로그램이나, 기타 프로그램을 봤을 때, 쉽게 데이터를 저장하고, 기록을 남길 수 있었으며, 도커의 사용이 없어도 충분히 이용이 가능한데. 그 단계까지 가기에는 아직 많이 어려웠던 거 같습니다.물론 지금 교육을 받는 단계라서 제대로 구현하기에는 실력이 부족할 수 있으나, 하지만, 그런 부분에 대해서 핑계를 대고 싶지는 않습니다. 좀 더 잘 구현하고 싶다는 욕심이 들었습니다.물론 AI는 도커 없이도 실행이 가능하다고 하지만, 그렇게 설명한 부분이 제게는 좀 어려웠고, 웹에서 이를 잘 구동하게끔 만드는 그런 부분들을 좀 더 잘 이해하고, 더 잘 만들 수 있었으면 좋았겠다 생각했습니다.
